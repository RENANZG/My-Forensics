File Systems Forensic Comparison

File System Deleted File Recovery Timestamps Journaling File Allocation Info Metadata Analysis File Carving Physical Data Recovery Tools (Examples)
NTFS Yes (with tools like ntfsundelete) Created, Modified, Accessed, MFT entry modified Yes (Transaction log) MFT record with allocation info MFT entries and $LogFile Yes (e.g., Photorec) Yes (with specialized hardware) Autopsy, EnCase, FTK
FAT32 Yes (limited due to lack of journaling) Created, Modified, Accessed No (No transaction log) FAT (File Allocation Table) Directory entries Yes (e.g., Photorec) Yes (with specialized hardware) Autopsy, TestDisk, Scalpel
ext4 Yes (with tools like extundelete) Created, Modified, Accessed, Changed Yes (Journaling) Block and inode level allocation Inodes and block groups Yes (e.g., Scalpel) Yes (with specialized hardware) Sleuth Kit, X-Ways Forensics, Scalpel
XFS Yes (with tools like scalpel) Created, Modified, Accessed Yes (Journaling) B+ trees and extent maps Metadata in inodes and extent records Yes (e.g., Scalpel) Yes (with specialized hardware) Sleuth Kit, X-Ways Forensics, Scalpel
Btrfs Yes (with tools like btrfs-undelete) Created, Modified, Accessed Yes (Copy-on-Write) B+ trees and extent maps Metadata and data checksums Yes (e.g., Photorec) Yes (with specialized hardware) Autopsy, The Sleuth Kit, btrbk
exFAT Yes (limited due to lack of journaling) Created, Modified, Accessed No (No transaction log) File Allocation Table (FAT) Directory entries Yes (e.g., Photorec) Yes (with specialized hardware) Autopsy, TestDisk, Scalpel
ZFS Yes (with tools like ZFSundelete) Created, Modified, Accessed Yes (Copy-on-Write, with ZIL and ARC) Object-based allocation Object sets and blocks Yes (limited, due to complex storage structure) Yes (with specialized hardware) Autopsy, The Sleuth Kit, ZFSundelete
